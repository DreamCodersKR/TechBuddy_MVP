# FLOWIT 서비스 정의서
> 작성일: 2026-03-19 | 버전: v1.0 | 작성자: DreamCoders
> 목적: 사업계획서 작성 레퍼런스 / 팀 내부 서비스 정의 기준 문서

---

## 1. 서비스 한 줄 정의

> **FLOWIT은 IT 성장을 이어가고 싶은 사람들을 위한 올인원 플랫폼이다.**
> 커뮤니티에서 연결하고 → 팀원모집으로 팀을 만들고 → PM툴로 협업하고 → AI 멘토링으로 성장한다.
> 이 모든 흐름이 하나의 서비스 안에서, 하나의 크레딧으로 이어진다.

---

## 2. 문제 정의 (왜 만드는가)

| 문제 | 현상 | 기존 해결책의 한계 |
|------|------|-----------------|
| **성장 단절** | 부트캠프·KDT 수료 후 성장 지원이 끊김. '취업 후 방향 모르겠다' 68% 응답 | 수료 후 연결 없음 |
| **도구 파편화** | IT인재 1인 평균 4~5개 도구 동시 사용 (Notion, Linear, ChatGPT, Slack 등) → 하루 1~2시간 Context Switching 낭비 | 통합 플랫폼 없음 |
| **AI 맥락 부재** | 범용 LLM 활용률 높으나 프로젝트 맥락 없이 사용 → 반복 설명, 얕은 답변 | 개인 컨텍스트 반영 안 됨 |
| **커뮤니티 파편화** | IT 취준생 커뮤니티가 카카오 오픈채팅·OKKY·링크드인에 분산 → 깊은 연결 없음 | 플랫폼이 달라 맥락 이어지지 않음 |

---

## 3. 핵심 가치 제안

**기존 서비스들이 각각 해결하는 것:**
- OKKY / 카카오 오픈채팅 → 커뮤니티만
- Notion / Linear / Jira → PM툴만
- ChatGPT / Claude → AI만

**FLOWIT이 다른 것:**
세 가지가 하나로 연결된다. 커뮤니티에서 만난 사람이 팀이 되고, 팀이 되면 PM툴이 생기고, 작업하다 막히면 그 컨텍스트 그대로 AI에게 질문한다. **전환 비용 0, 맥락 손실 0.**

---

## 4. 서비스 구조 (네비게이션)

```
FLOWIT
├── 커뮤니티
│   ├── 게시판 (자유게시판)
│   ├── 팀원모집 ([프로젝트] / [스터디] 태그)
│   └── 아고라 (크레딧 기반 Q&A)
│
└── 프로젝트 (PM툴)
    ├── 태스크
    ├── 스프린트
    └── 칸반보드
```

> AI 멘토링은 독립 탭이 아닌 **서비스 전반에 임베드** — 게시판 작성 중, 아고라 답변 중, 태스크 작업 중 어디서든 호출 가능

## 5. 기능 상세 정의

### 5-1. 게시판 (자유게시판)

**목적:** IT 커뮤니티 구성원 간 자유로운 정보 공유, 취업 후기, 기술 토론

**특징:**
- 단일 자유게시판으로 운영 (초기)
- 게시글에 좋아요·추천 → 뱃지 점수 반영
- 추천수 기반 인기글 노출 (에펨코리아 방식)
- 유저 활동이 쌓이면 "기술 토론 / 취업후기 / 자유" 카테고리 분리 검토 (유저 1,000명 이후)

---

### 5-2. 팀원모집

**목적:** 사이드 프로젝트·스터디 팀 구성, PM툴 연계의 시작점

**구조:**
- 단일 탭으로 통합 운영
- 게시글 작성 시 **[프로젝트] / [스터디]** 태그 선택 필수
- 필터 검색으로 구분 가능

**태그별 차별화:**

| | [프로젝트] | [스터디] |
|--|-----------|---------|
| 대상 | 사이드프로젝트·해커톤 팀원 | 알고리즘·CS·포트폴리오 스터디 |
| PM툴 연계 버튼 | ✅ "팀 프로젝트 만들기 →" | ❌ (심플 게시판 형태) |
| 권장 플랜 | Pro 이상 | Free 가능 |

**핵심 플로우:**
```
팀원모집 [프로젝트] 게시글 작성
→ 팀원 모집 완료
→ "팀 프로젝트 만들기" 버튼 클릭
→ 프로젝트 탭에 칸반/스프린트 자동 생성
→ 팀원들 초대 → PM툴 바로 시작
```
> 이 흐름이 FLOWIT의 핵심 유저 여정이자 타 서비스와의 가장 강력한 차별화 지점

---

### 5-3. 아고라 (크레딧 기반 Q&A)

**목적:** 크레딧을 매개로 고품질 Q&A 생태계 구성

**작동 방식:**
- 네이버 지식인 + Stack Overflow 현상금 방식
- 질문자가 자유롭게 크레딧을 걸고 질문 등록 (최솟값 없음)
- 채택된 답변자에게 걸린 크레딧 전액 지급
- UI soft hint: "💡 10크레딧 이상 스택 시 답변 품질이 높아집니다" (강제 아님)

**크레딧 경제 연결:**
- AI 멘토링과 동일한 크레딧 사용
- 답변으로 크레딧 획득 → AI 멘토링에 바로 사용 가능
- "AI로 못 풀면 사람한테, 사람도 막히면 AI 전문가 모드로" — 하나의 크레딧 생태계 안에서 선택

**접근 권한:**
- Free 유저도 아고라 참여 가능 (100크레딧 보유)
- 단, 크레딧 소진 시 추가 충전 또는 유료 플랜 유도

---

### 5-4. PM툴 (프로젝트 탭)

**목적:** 팀원모집에서 결성된 팀의 실무 협업 도구

**구성:**
- 태스크: 할 일 목록, 담당자 지정, 마감일
- 스프린트: 기간 단위 목표 관리
- 칸반보드: 진행 상태 시각화 (To Do / In Progress / Done)

**칸반 상태:**

| 상태 | 설명 |
|------|------|
| Todo | 시작 전 |
| In Progress | 진행 중 |
| Done | 완료 |
| **Help** ⭐ | 막힘 → AI 멘토링 or 아고라 자동 연결 |

**Help 상태 플로우 (핵심 차별화):**

```
태스크 작업 중 막힘 발생
→ 상태를 [Help] 로 변경
→ FLOWIT 팝업 등장:
   ① "AI 멘토링으로 해결하기"  → 태스크 컨텍스트 자동 포함 + AI 호출 (크레딧 소모)
   ② "아고라에 질문 올리기"    → 태스크 내용 자동 첨부 + 크레딧 걸기
→ 해결 완료 시 Help → Done 전환
```

> **컨텍스트 손실 제로**: 막힌 내용을 따로 복붙할 필요 없이 태스크 내용이 자동으로 AI/아고라에 첨부됨
> **사업계획서 한 문장 표현**: "막히면 Help 누르세요. AI한테 물어볼지, 사람한테 물어볼지만 고르면 됩니다."

**차별화:**
- Jira는 복잡해서 취준생이 쓰기 어렵고, Notion은 PM툴로 쓰기 아쉬움
- FLOWIT PM툴은 **커뮤니티에서 자연스럽게 유입**되는 구조 → 온보딩 허들 최소화
- Help 상태로 PM툴 → AI멘토링/아고라 **3방향 연결** 완성

---

### 5-5. AI 멘토링 (멀티모델 라우팅)

**목적:** IT 성장에 특화된 맥락 기반 AI 코칭

**구조:**
- 2축 라우팅: 구독 등급(수직) × Task 유형(수평)
- 기본 / 심화 / 전문가 3단계 티어
- GPT, Claude, Gemini, Grok 멀티모델 자동 라우팅

**서비스 내 임베드 위치:**
- 게시판 글 작성 중 → "AI 초안 작성 도움 받기"
- 아고라 질문 전 → "AI한테 먼저 물어보기"
- PM툴 태스크 카드 → "이 작업 AI에게 질문"
- 독립 AI 채팅 화면에서도 사용 가능

## 6. 뱃지 시스템

**목적:** 커뮤니티 활성화, 유료 전환 유도, 소속감 강화

### 뱃지 종류 및 지급 기준

| 카테고리 | 뱃지 이름 | 지급 조건 |
|---------|---------|---------|
| **신규** | 🌱 새싹 | 회원가입 즉시 |
| **활동** | ⚡ 활동가 | 활동점수 100점 달성 (일일 출석 1점 + 게시글 추천 5점 등) |
| **활동** | 🔥 고수 | 활동점수 500점 달성 |
| **활동** | 👑 레전드 | 활동점수 2,000점 달성 |
| **구독** | 💎 Pro | Pro 플랜 구독 중 |
| **구독** | ✨ 프리미엄 | 프리미엄 플랜 구독 중 |
| **로열티** | 🏅 장기 멤버 | 구독 3개월 이상 연속 유지 |
| **로열티** | 🥇 충성 멤버 | 구독 12개월 이상 연속 유지 |
| **소속** | [부트캠프 로고] | Enterprise 계약 부트캠프 소속 유저 전용 |

### 뱃지 시스템 비즈니스 효과

- **유료 전환 유도**: Pro·프리미엄 뱃지가 커뮤니티에 노출 → "나도 저 뱃지 달고 싶다" 심리
- **Enterprise B2B 유인**: 부트캠프 로고 뱃지 → 수강생 프로필에 소속 브랜딩 → 부트캠프 홍보 효과
- **리텐션**: 로열티 뱃지 → 구독 유지 동기 강화
- **활동 유지**: 활동 기반 뱃지 → 일일 출석·게시글 작성 유도

### 뱃지 표시 방식
- 프로필 페이지에서 사용자가 **대표 뱃지** 직접 설정 (에펨코리아 방식)
- 게시판 게시글·댓글 작성자 옆에 대표 뱃지 표시
- 아고라 답변자 프로필에도 뱃지 노출 → 신뢰도 지표 역할

## 7. 크레딧 경제 구조

**핵심 원칙: 하나의 크레딧이 AI와 사람을 모두 연결한다.**

```
크레딧 획득 경로
├── 플랜 구독 (매월 자동 지급: Free 100 / Pro 3,000 / 프리미엄 5,000)
├── 아고라 답변 채택 (질문자가 건 크레딧 전액 수령)
└── 추가 충전 (₩5,500 / ₩22,000 / ₩38,500)

크레딧 소비 경로
├── AI 멘토링 (기본 1크레딧 / 심화 10크레딧 / 전문가 50크레딧)
└── 아고라 질문 (자유롭게 크레딧 스택)
```

**선순환 구조:**
```
좋은 답변 작성 → 크레딧 획득 → AI 멘토링에 사용 → 더 성장 → 더 좋은 답변
```

## 8. 세 기능의 연결 플로우 (핵심 차별화)

### 시나리오: 부트캠프 수료생 김민준의 하루

```
① 아침 출석 체크 → 활동점수 1점 + 게시판 인기글 확인
   (게시판)

② 사이드 프로젝트 팀원 구하기 → 팀원모집 [프로젝트] 게시글 작성
   "React 포트폴리오 프로젝트 팀원 2명 모집"
   (팀원모집)

③ 팀 결성 완료 → "팀 프로젝트 만들기" 클릭
   → 프로젝트 탭에 칸반보드 자동 생성, 팀원 초대
   (PM툴)

④ 태스크 작업 중 막히는 부분 발생
   → 태스크 상태를 [Help] 로 변경
   → FLOWIT 팝업: "AI에게 물어볼까요?" or "아고라에 질문 올릴까요?"
   → 태스크 컨텍스트 자동 첨부 → 따로 설명 필요 없음
   (PM툴 Help → AI 멘토링 or 아고라)

⑤ AI 멘토링 선택 시: Sonnet 심화 AI 호출 (10크레딧 차감)
   → 프로젝트 맥락 기반 맞춤 답변 → Help → Done 전환
   (AI 멘토링)

⑥ 아고라 선택 시: 크레딧 20개 스택 → 현직자 답변 채택
   → 크레딧 받은 현직자가 AI 멘토링에 다시 사용
   (아고라 → AI 선순환)
```

**이것이 FLOWIT의 차별화:** OKKY에서 질문하고 → Notion에서 정리하고 → ChatGPT에 붙여넣는 3단계를 FLOWIT 하나에서 끝낸다.

## 9. 타겟 유저 페르소나

### B2C 주력 타겟

| 페르소나 | 특징 | FLOWIT에서 주로 사용하는 기능 |
|---------|------|--------------------------|
| **부트캠프 수료생** (25~30세) | 취업 준비 중, 포트폴리오 사이드 프로젝트 필요 | 팀원모집 → PM툴 → AI 멘토링 |
| **IT 취준생** (22~28세) | CS 스터디·알고리즘 스터디 필요 | 팀원모집 [스터디] → 아고라 |
| **1인 개발자** (25~35세) | 사이드 프로젝트 진행 중, AI 활용 높음 | AI 멘토링 → PM툴 |

### B2B 타겟

| 고객사 | 계약 형태 | FLOWIT 활용 |
|--------|---------|------------|
| **부트캠프·KDT 교육기관** | Enterprise | 수강생 전원 Pro 제공, 기관 로고 뱃지 |
| **소규모 IT 스타트업** | Enterprise | 팀 PM툴 + AI 멘토링 통합 |

## 11. 미결 사항 (추후 논의 필요)

| 항목 | 현재 상태 | 논의 필요 내용 |
|------|---------|-------------|
| 게시판 카테고리 확장 | 자유게시판 단일 | 유저 1,000명 이후 분리 기준 |
| 아고라 채택 기준 | 질문자가 채택 | 일정 기간 후 미채택 시 자동 처리 방법 |
| 팀원모집 탭 분리 시점 | 통합 운영 | 유저 볼륨 기준점 결정 필요 |
