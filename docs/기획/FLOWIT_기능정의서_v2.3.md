## 1. 공통 - 네비게이션

### 상단 GNB (Global Navigation Bar)
| 항목 | 내용 | 구현상태 |
|------|------|---------|
| 로고 | FLOWIT (클릭 시 `/` 이동) | ✅ 완료 |
| 메뉴 | 커뮤니티 \| 프로젝트 | ✅ 완료 |
| 검색 | 전체 검색창 | ✅ 완료 |
| 알림 | 벨 아이콘 + 미읽음 배지 (30초 폴링) | ✅ 완료 |
| 인증 | 비로그인: 로그인 버튼 / 로그인: 프로필 드롭다운 | ✅ 완료 |

### 프로필 드롭다운 (로그인 시)
- 마이페이지 (`/mypage`)
- 내 게시글 (`/mypage/posts`)
- 북마크 (`/mypage/bookmarks`)
- 설정 (`/settings`)
- 로그아웃

### 커뮤니티 서브탭
```
게시판 | 아고라 | 팀원 모집
```

### PM툴 좌측 사이드바 (워크스페이스 진입 시)
```
FLOWIT                        ← 로고/홈
────────────────────
내 Task                       ← 전체 워크스페이스에서 내게 배정된 Task
────────────────────
내 워크스페이스 ▼
  📁 포트폴리오 웹             ← [프로젝트]
  📚 알고리즘 스터디            ← [스터디룸]
  + 새 워크스페이스
────────────────────
(선택된 워크스페이스가 프로젝트일 때)
  └ 대시보드
  └ 칸반 보드
  └ 태스크 목록
  └ 스프린트
  └ 멤버
  └ 산출문서
  └ 설정
────────────────────
(선택된 워크스페이스가 스터디룸일 때)
  └ 대시보드
  └ 커리큘럼 (주차별)
  └ 과제 현황판
  └ 벌금 관리
  └ 자료 모음
  └ 멤버
  └ 설정
────────────────────
AI 멘토                       ← 하단 고정
```

### 미니 프로필 팝업 (공통 컴포넌트)

> **트리거**: 커뮤니티 게시판/상세/댓글, 아고라 목록/상세에서 작성자 **닉네임 또는 아바타 클릭**

| 항목 | 내용 |
|------|------|
| 표시 조건 | 닉네임이 설정된 사용자만 (nickname != null) |
| 위치 | 클릭한 요소 기준 자동 계산 (화면 넘침 방지) |
| 닫힘 조건 | 팝업 외부 클릭 또는 스크롤 시 |

**팝업 표시 정보**

| 항목 | 내용 |
|------|------|
| 아바타 | 프로필 이미지 (없으면 닉네임 첫 글자) |
| 닉네임 | 굵게 표시 |
| 이름 | 실명 표시 |
| 레벨 | Lv.N (레벨에 따라 색상 구분: Lv.1~2 초록, Lv.3~4 파랑, Lv.5+ 주황) |
| 자기소개 | bio (최대 2줄) |
| 기술스택 | 최대 5개 표시, 초과 시 +N |

**메뉴 구성**

| 메뉴 | 동작 | 조건 |
|------|------|------|
| 포트폴리오 보기 | `/portfolio/:nickname` 이동 | `portfolioPublic = false`이면 비활성화 + "비공개" 표시 |
| 작성 글 보기 | `/community?author=:nickname` 이동 | 항상 표시 |
| 신고하기 | 신고 처리 | **자기 자신이면 숨김** |

**적용 위치**: 커뮤니티 목록, 커뮤니티 상세(게시글 작성자 + 댓글 작성자), 아고라 목록, 아고라 상세(질문 작성자 + 답변 작성자)

## 2. 메인 홈 (/)

커뮤니티 중심 허브 페이지. PM툴·AI멘토링은 배너로만 소개.

### 섹션 구성 (상단 → 하단)

| 순서 | 섹션명 | 내용 | 기준 | 구현상태 |
|------|--------|------|------|---------|
| 1 | 서브탭 | 게시판 \| 아고라 \| 팀원 모집 | 커뮤니티 섹션 탭 | ✅ |
| 2 | 소개 배너 | FLOWIT 서비스 소개 | - | ✅ |
| 3 | 실시간 인기글 | 게시판 인기글 5개 리스트 | **좋아요 수 / 최근 7일** | ✅ |
| 4 | 실시간 인기 아고라 | 아고라 인기 질문 리스트 | **조회수 기준** / 💎 현상금 표시 | ✅ |
| 5 | 새로운 인사이트를 얻어가세요 | **아티클만** (게시판 글 제외) | 최신순 / 더보기 링크 | ✅ |
| 6 | 현재 답변을 기다리고 있어요 | 아고라 **미채택** 질문 리스트 | 최신순 / 더보기 링크 | ✅ |
| 7 | 프로젝트 기능 소개 배너 | PM툴 기능 소개 CTA | - | ✅ |
| 8 | 지금, 함께 할 개발자를 찾고 있어요 | 팀원모집 최신 모집글 | 최신순 / 더보기 링크 | ✅ |
| 9 | 푸터 | 사업자정보, 이용약관, 개인정보처리방침 | - | ✅ |

## 3. 커뮤니티 - 게시판

### 페이지 구조
```
/community           # 게시판 (기본 탭)
```

### 카테고리 탭
```
전체 | 자유 | 취업·진로 | 아티클
```

| 카테고리 | 작성 권한 | 설명 |
|----------|----------|------|
| 자유 | 모든 회원 | 자유 주제 |
| 취업·진로 | 모든 회원 | 취업, 이직, 커리어 관련 |
| 아티클 | **FLOWIT 관리자(admin)만** | 팀 발행 아티클 |

### 작성자 상호작용
- 게시글 목록 작성자 닉네임 클릭 → **미니 프로필 팝업** (→ 섹션 1 공통 참조)
- 게시글 상세 작성자 아바타/닉네임 클릭 → **미니 프로필 팝업**
- 댓글 작성자 아바타/닉네임 클릭 → **미니 프로필 팝업**

### 인기글 기준
- 기준: **좋아요 수**
- 기간: **최근 7일**
- 표시: 상단 "실시간 인기글" 섹션에 5개

### 아티클 전용 형식
| 필드 | 타입 | 필수 여부 |
|------|------|----------|
| 썸네일 이미지 | 이미지 업로드 | 필수 |
| 제목 | 텍스트 | 필수 |
| 본문 | 리치텍스트 에디터 | 필수 |
| 태그 | 멀티 태그 입력 | 선택 |
| 작성자 | **"FLOWIT팀" 고정** | 자동 |
| 읽기 예상 시간 | 글자 수 기반 자동 계산 | 자동 |
| 시리즈 묶음 | 기존 시리즈 선택 or 새 시리즈 생성 | 선택 |

## 4. 커뮤니티 - 팀원모집

### 페이지 구조
```
/recruit             # 팀원모집 목록
/recruit/write       # 모집글 작성
/recruit/:id         # 모집글 상세
```

### 유형 탭 & 상태 필터
```
[전체 | 프로젝트 | 스터디]          ← 유형 탭
[전체 | 모집중 | 마감]              ← 상태 필터 (기본값: 전체)
```
> ✅ v2.0 Hotfix(DRE-225): 상태 필터 기본값 수정 (open → all), 필터 순서 정렬

### 포지션 필터 태그
```
프론트엔드 | 백엔드 | 풀스택 | 디자이너 | 기획자 | 마케터 | DevOps | iOS | Android
```
- 다중 선택 불가 (단일 선택 토글)

### 모집글 카드 구성
```
[유형 뱃지] 프로젝트/스터디룸명         [마감] (마감 시)
제목
설명 한 줄
포지션 태그
X/N명  ·  마감일  ·  작성자
```

### 작성 조건 및 진입점
| 항목 | 내용 |
|------|------|
| 작성 진입점 | 팀원모집 목록 > "모집글 작성" 버튼 |
| 작성 조건 | 로그인 필요 (PM툴 프로젝트/스터디룸 연결 선택사항) |

### 지원 플로우 ✅ (DRE-143 완료)
1. 지원자: "지원하기" 클릭
2. 자기소개 메시지 입력 모달 팝업
3. 제출 → 작성자에게 알림
4. 작성자: 지원자 목록 확인 (작성자만 열람)
5. 수락 클릭 → PM툴 해당 프로젝트/스터디룸에 멤버 자동 초대
6. 지원자: 지원 상태 표시 **(검토중 / 수락 / 거절)**

### 지원 상태 표시 (지원자 본인)
- 목록 카드: "지원 완료" 배지
- 상세 페이지: 현재 상태 (검토중 / 수락됨 / 거절됨)

### 마감 처리
| 방식 | 내용 |
|------|------|
| 마감일 자동 마감 | 설정한 마감일 도래 시 자동 마감 |
| 수동 마감 | 작성자가 "마감하기" 버튼으로 직접 마감 |
| 인원 초과 자동 마감 | 없음 (작성자 판단) |

### 상세 페이지 권한
| 대상 | 보이는 것 |
|------|----------|
| 일반 방문자 | 모집 내용 + 지원하기 버튼 |
| 작성자 | 지원자 목록 + 수락/거절 버튼 |
| 지원자 본인 | 본인 지원 상태 |

## 5. 커뮤니티 - 아고라

### 페이지 구조
```
/agora               # 아고라 질문 목록
/agora/write         # 질문 작성
/agora/:id           # 질문 상세 (답변 목록 포함)
```

### 핵심 개념
> **크레딧 현상금 기반 Q&A** — 네이버 지식인 + Stack Overflow 현상금 방식
> 모든 질문에 크레딧 베팅 필수. 채택 시 현상금 자동 지급.

### 탭 구성
```
전체 | 미채택 | 채택완료
```

### 정렬 옵션
```
최신순 | 현상금 높은순 | 답변 많은순
```

### 작성자 상호작용
- 질문 목록 작성자 닉네임 클릭 → **미니 프로필 팝업** (→ 섹션 1 공통 참조)
- 질문 상세 작성자 닉네임 클릭 → **미니 프로필 팝업**
- 답변 작성자 닉네임 클릭 → **미니 프로필 팝업**

### 현상금 설정 (질문 작성 시)
| 항목 | 내용 |
|------|------|
| 필수 여부 | **필수** (최소 1크레딧) |
| 범위 | 최소 1cr ~ 본인 잔액 전부 |
| 입력 방식 | 프리셋 버튼 + 직접 입력 `[1cr] [10cr] [50cr] [100cr] [직접입력]` |

### 질문 상세 페이지
| 항목 | 내용 |
|------|------|
| 채택 버튼 | 질문 작성자만 표시 (답변 카드마다) |
| 본인 답변 | 질문 작성자는 자기 질문에 답변 불가 |
| 채택된 답변 | 상단 고정 표시 |
| 채택 후 | 현상금 크레딧 답변자에게 자동 지급 |

### 수정/삭제 정책
| 상황 | 가능 여부 | 크레딧 처리 |
|------|----------|-----------|
| 답변 0개일 때 | 수정/삭제 가능 | 삭제 시 현상금 전액 환불 |
| 답변 1개 이상 | 수정/삭제 불가 | - |
| 채택 완료 후 | 삭제 불가 | - |

## 6. PM툴 - 개요 및 플랜 제한

### 핵심 차별점
> **Help 상태** — Task를 Help로 전환하면 태스크 컨텍스트 자동 첨부 → AI멘토링 or 아고라 즉시 연결

### 플랜별 기능 제한

| 항목 | Free | Pro ★ | 프리미엄 | 구현 상태 |
|------|------|-------|---------|---------|
| **프로젝트 생성** | 1개 | 무제한 | 무제한 | ✅ 백엔드 구현 완료 |
| **스터디룸 생성** | 1개 | 무제한 | 무제한 | ✅ 백엔드 구현 완료 |
| **팀원 초대** | 제한없음* | 5명/워크스페이스 | 무제한 | ⏳ DRE-238 (Backlog) |
| **Task 수** | 20개/워크스페이스 | 무제한 | 무제한 | ❌ 미구현 |
| **Sprint** | 없음 | ✅ | ✅ | ❌ 플랜 제한 미구현 |
| **칸반보드** | 기본 5컬럼 | 기본 5컬럼 | ✅ + 커스텀 컬럼 | ⏳ DRE-237 (Backlog) |
| **파일 첨부** | 없음 | 10MB/파일 | 50MB/파일 | ❌ 미구현 |

## 14. PM툴 - 스터디룸 (재정의)

### 핵심 철학
> "To Do → Done"이 아닌 **"N명이 이번 주 과제를 했는가(O/X)"**
> 분업 도구가 아닌 **강제성 + 루틴 형성 도구**

### 기존 대비 변경점
| 항목 | AS-IS (v1.3) | TO-BE (v2.0) |
|------|-------------|-------------|
| 레이아웃 | 프로젝트와 동일 (칸반보드) | **스터디 전용 레이아웃** |
| 주요 기능 | 칸반보드, Sprint | 커리큘럼, 과제 현황판, 벌금 관리 |

### 14-1. 주차별 커리큘럼

**개념**: 스터디 전체 일정을 주차 단위로 구성. Sprint 컴포넌트를 재활용하되 명칭/UX를 스터디에 맞게 변형.

**화면 구성**
```
[커리큘럼]
─────────────────────────────────
Week 1  2026-04-07 ~ 2026-04-13  [진행중]
  └ 과제: 알고리즘 - DP 기초 5문제 풀기
  └ 인증 방식: Github PR 링크 제출
Week 2  2026-04-14 ~ 2026-04-20  [예정]
  └ 과제: 알고리즘 - BFS/DFS 5문제
  └ 인증 방식: Github PR 링크 제출
[+ 주차 추가]
─────────────────────────────────
```

**주차 필드**
| 필드 | 내용 |
|------|------|
| 주차명 | 자동 (Week 1, Week 2...) 또는 직접 입력 |
| 기간 | 시작일 ~ 종료일 (기본 7일) |
| 과제 내용 | 텍스트 (마크다운 지원) |
| 인증 방식 | Github PR / 블로그 링크 / 파일 업로드 / 텍스트 인증 |
| 상태 | 예정 / 진행중 / 완료 |

**접근 권한**
- 스터디장(관리자)만 주차 생성/수정/삭제 가능
- 멤버: 읽기 가능

---

### 14-2. 과제 인증 현황판 (핵심 기능)

**개념**: 칸반보드를 완전 대체하는 스터디 특화 뷰.
멤버 × 주차 테이블로 각자의 과제 제출 여부를 한눈에 확인.

**화면 구성**
```
          | 홍길동 | 이영희 | 박철수 | 김민지 |
Week 1    |   ✅  |   ✅  |   ❌  |   ✅  |
Week 2    |   ✅  |   ⏳  |   ⏳  |   ⏳  |
Week 3    |   -   |   -   |   -   |   -   |

✅ 제출완료  ❌ 미제출  ⏳ 진행중  - 예정
```

**제출 플로우**
1. 멤버가 현황판에서 자신의 셀 클릭
2. 인증 방식에 따라 링크/파일/텍스트 입력 모달 표시
3. 제출 → 셀 상태 `⏳ → ✅` 변경
4. 스터디장이 검토 후 최종 승인 가능 (선택 옵션)

**셀 클릭 시 상세 보기**
- 멤버의 제출 내역 (링크, 파일, 텍스트)
- 제출 시각
- 스터디장 코멘트 입력 가능

**마감일 처리**
- 주차 종료일 자정에 미제출 → `❌` 자동 처리
- 자동 결석 처리 시 벌금 자동 누적 (설정된 경우)

### 14-3. 벌금/참여율 관리 (킬러 기능)

**개념**: IT 스터디 90%가 운영하는 보증금/벌금 제도를 FLOWIT 크레딧 기반으로 디지털화.
카카오톡 공지방 + 엑셀 조합을 대체하는 핵심 기능.

> 실제 돈이 아닌 **FLOWIT 크레딧(cr)** 기반 에스크로 시스템

---

#### 전체 플로우

```
[스터디장] 벌금 규칙 설정
    └ 보증금: 500cr / 미제출: 50cr / 결석: 200cr / 기간: 4/7 ~ 6/29
    └ [벌금 기능 활성화] 버튼 클릭
         ↓
[시스템] 전체 멤버에게 알림 발송
    └ "알고리즘 스터디 벌금 규칙이 설정되었습니다"
    └ 보증금: 500cr / 미제출: 50cr / 결석: 200cr
    └ [동의] [거절]
         ↓
[전원 동의 완료] → 각 멤버 크레딧에서 보증금 즉시 차감 → 스터디 공용 Pool 에스크로
         ↓
[기간 중] 규칙 위반 시 Pool 내에서 자동 벌금 처리
         ↓
[기간 종료] Pool 잔액을 참여 멤버 수 N분의1로 자동 분배
```

---

#### 규칙 설정 (스터디장만)

| 항목 | 내용 | 필수 여부 |
|------|------|---------|
| 보증금 | 멤버별 예치 크레딧 (예: 500cr) | 필수 |
| 미제출 벌금 | 과제 미제출 시 차감 (예: 50cr) | 필수 |
| 결석 벌금 | 모임 불참 시 차감 (예: 200cr) | 선택 |
| 지각 벌금 | 지각 시 차감 (예: 20cr) | 선택 |
| 적용 기간 | 시작일 ~ 종료일 | 필수 |

> 벌금 합계가 보증금을 초과할 수 없음. 보증금 한도 내에서만 차감 (마이너스 없음)

---

#### 동의 절차

| 상태 | 조건 |
|------|------|
| 기능 활성화 | **전원 동의** 시에만 활성화 |
| 1명이라도 거절 | 기능 OFF 유지, 스터디장에게 알림 |
| 크레딧 부족 멤버 | 보증금 예치 불가 → 자동 거절 처리 → 스터디장 알림 |
| 동의 기한 | 설정 후 **24시간** 내 미응답 → 자동 거절 처리 |

---

#### 공용 Pool 에스크로

- 전원 동의 완료 시 각 멤버 크레딧에서 보증금 즉시 차감
- 시스템이 스터디 공용 Pool로 보관 (멤버 접근 불가)
- 규칙 위반 감지(자동) or 스터디장 수동 입력 시 Pool 내에서 차감

#### 기간 종료 시 자동 정산

1. 기간 종료일 자정 → Pool 잔액 확정
2. 잔액 ÷ 참여 멤버 수 = 1인당 환급 크레딧
3. 각 멤버 계정에 자동 지급
4. 전체 멤버에게 정산 완료 알림 발송

```

이영희: 312cr 지급 완료
박철수: 312cr 지급 완료
김민지: 312cr 지급 완료
```

## 15. AI 멘토링 - 개요 및 진입점

### 핵심 개념
> **작업 맥락을 끊지 않고 바로 도움받기** — Task 컨텍스트 자동 첨부 + 멀티모델 라우팅 + SSE 실시간 스트리밍

### 진입점 (4가지)

| 진입점 | 설명 | 구현상태 |
|--------|------|---------|
| ① 우측 플로팅 버튼 💬 | 모든 페이지에서 즉시 접근 | ✅ |
| ② 좌측 워크스페이스 사이드바 하단 | 워크스페이스 내 빠른 접근 | ✅ |
| ③ Task Help 상태 | 칸반에서 Help 클릭 → Task 컨텍스트 자동 첨부 | ✅ |
| ④ Task 상세 AI 멘토 버튼 | Task 상세 화면 우측에 AI 채팅 패널 | ⏳ DRE-226 |

### Task 상세에서 AI 멘토 패널 (DRE-226)
```
┌───────────────────────────────┬─────────────────────┐
│  Task 상세                    │  AI 멘토 💬        X │
│  FLOW-001 버그 수정            ├─────────────────────┤
│                               │  Task 컨텍스트 첨부:  │
│  설명:                         │  FLOW-001 버그 수정   │
│  [마크다운 뷰어]                │                     │
│                               │  [대화 내용]          │
│  상태: 진행중                  │                     │
│  담당자: 홍길동                │  ────────────────   │
│                               │  [입력창]      [전송] │
└───────────────────────────────┴─────────────────────┘
```
- Task 상세 컨텍스트(제목, 설명, 상태) 자동 첨부
- AI 멘토 패널 닫기(X) → Task 상세만 표시

### SSE 스트리밍 ✅ (DRE-147 완료)
- NestJS `@Post()` + `res.write()` 방식의 AsyncGenerator SSE

- 실시간 타이핑 효과 (ChatGPT 방식)

### 페이지 구조


### 포트폴리오 공개 페이지 (`/portfolio/:nickname`)

> **목적**: 외부 공유 가능한 개인 활동 포트폴리오. SNS/링크드인 공유 → 바이럴 유입.

**SEO / 공유**
| 항목 | 내용 |
|------|------|
| 렌더링 | SSR (서버사이드 렌더링) |
| OG 메타태그 | og:title, og:description, og:image, og:url 자동 생성 |
| 비공개 처리 | `portfolioPublic = false` → 404 반환 |
| 인증 | 불필요 (완전 공개) |

**페이지 섹션 구성**

| 섹션 | 내용 | 표시 조건 |
|------|------|---------|
| 프로필 헤더 | 아바타, 닉네임, 이름, 레벨, 기술스택, bio | 항상 |
| 활동 요약 | 총 TIL 수, 연속 스트릭, 아고라 채택 수, 프로젝트 수 | `sections.stats` |
| TIL 잔디 | 52주 × 7일 활동 히트맵 (ActivityHeatmap) | `sections.grass` |
| 프로젝트 | Verified 프로젝트 목록 (초록 배지) | `sections.projects` |
| 뱃지 | 획득한 뱃지 목록 | `sections.badges` |
| 최근 TIL | 공개/팀공개 TIL 최근 5개 | `sections.recentTILs` |
| 푸터 | "Powered by FLOWIT" 링크 | 항상 |

**Verified Project 조건** (실시간 계산, DB 필드 없음)
- 완료된 스프린트 ≥ 2개
- 팀 멤버 ≥ 2명
- 산출문서 ≥ 1개

### 포트폴리오 설정 (`/settings` 내 섹션)

| 항목 | 내용 |
|------|------|
| 공개 여부 | 토글 (기본값: 공개) |
| 섹션 선택 | 체크박스 5개 (활동요약/잔디/프로젝트/뱃지/최근TIL) |
| 링크 복사 | `https://flowit.co/portfolio/:nickname` 클립보드 복사 |
| 미리보기 | 새 탭에서 `/portfolio/:nickname` 열기 |

### API

| 메서드 | 엔드포인트 | 인증 | 설명 |
