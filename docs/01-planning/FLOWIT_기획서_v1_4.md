## 1. 프로젝트 개요

### 1.1 서비스명
**FLOWIT** (플로잇)

> 이전 서비스명: TechBuddy
> 변경 사유: 로고 디자인 및 브랜딩 용이성

### 1.2 서비스 정의
IT 부트캠프 학생과 취준생, 주니어 개발자를 위한 **커뮤니티 기반 성장 플랫폼**

### 1.3 핵심 가치
- **성장**: 멘토링과 프로젝트를 통한 실력 향상
- **연결**: 동료 및 멘토와의 네트워킹
- **실용성**: 초보자 친화적인 프로젝트 관리 도구
- **커뮤니티**: 교육기관 및 주제별 맞춤 정보 공유

### 1.4 차별화 포인트
1. **부트캠프 학생 특화**: 일반 개발자 커뮤니티와 달리 초보자 중심
2. **올인원 플랫폼**: 커뮤니티 + 멘토링 + 프로젝트 관리 + 교육기관 홍보
3. **초보자용 PM 도구**: Linear/Jira의 복잡함 없이 쉬운 프로젝트 관리
4. **검증된 유입 채널**: 부트캠프 강사 출신 팀원을 통한 직접 영업

## 2. 문제 정의 및 솔루션

### 2.1 타겟 사용자의 Pain Points

#### 부트캠프 학생/취준생
```
❌ 문제 1: 실무 경험 부족
   → 포트폴리오용 프로젝트 필요하지만 막막함

❌ 문제 2: 멘토 찾기 어려움
   → 주변에 물어볼 시니어 개발자가 없음

❌ 문제 3: 프로젝트 관리 도구 진입장벽
   → Jira, Linear 너무 복잡하고 팀원도 안 씀

❌ 문제 4: 정보 부족
   → 취업 정보, 실무 팁을 어디서 얻어야 할지 모름

❌ 문제 5: 교육기관 정보 파편화
   → 각 부트캠프 정보가 흩어져 있어 비교 어려움
```

#### 주니어 개발자
```
❌ 문제 1: 성장 정체
   → 혼자 공부하는데 한계, 방향성 불명확

❌ 문제 2: 네트워킹 부족
   → 사이드 프로젝트 같이 할 동료 찾기 어려움

❌ 문제 3: 커리어 고민
   → 이직, 기술 스택 선택 등 조언 필요
```

### 2.2 FLOWIT의 솔루션

```
✅ 솔루션 1: 주제별 커뮤니티 + Q&A
   → 자유게시판, Q&A, 팀원 모집 등 맞춤 정보

✅ 솔루션 2: 멘토링 매칭 시스템
   → 인증된 멘토와 쉽게 연결

✅ 솔루션 3: 초보자용 프로젝트 관리
   → 배우기 쉬운 칸반보드 + 협업 도구
   → Help 상태로 막힌 부분 즉시 요청

✅ 솔루션 4: 검증된 유입 채널
   → 부트캠프와 파트너십을 통한 안정적 유저 확보
```

## 3. 타겟 사용자

### 3.1 Primary Target (주요 타겟)

#### 부트캠프 학생
- 코딩 부트캠프 수강 중이거나 수료한 학생
- 나이: 20-35세
- 특징: IT 커리어 전환, 비전공자 많음
- 니즈: 취업 준비, 포트폴리오, 멘토링

#### IT 취준생
- IT 분야 취업 준비 중
- 포트폴리오 프로젝트 필요
- 면접 준비 및 정보 교환

### 3.2 Secondary Target (보조 타겟)

#### 주니어 개발자 (0-3년 차)
- 실무 경험 쌓고 있는 단계
- 사이드 프로젝트 관심 있음
- 커리어 성장 고민

#### 시니어 개발자/멘토
- 멘토링에 관심 있는 경력 개발자
- 지식 공유 및 보람
- 추후 유료 멘토링 수익 가능

## 4. 서비스 플로우 ⭐ v1.4 변경

### 4.1 플로우 변경 사항

#### 기존 (v1.3 - Linear 스타일)
```
[랜딩 페이지] → [시작하기 클릭] → [로그인/회원가입] → [앱 메인 (/app/*)]
                                                              │
                                                    ┌─────────┴─────────┐
                                                    ▼                   ▼
                                              [커뮤니티]           [프로젝트 관리]
                                              (사이드바 내비게이션)
```

#### 변경 (v1.4 - 커뮤니티 베이스)
```
[메인 페이지 = 커뮤니티] ← 바로 진입
         │
         ├─ 비로그인: 콘텐츠 열람 가능, 작성/참여 시 로그인 유도
         │
         └─ 로그인: 모든 기능 이용 가능
                    │
                    └─ [프로젝트] 클릭 → 새 탭에서 프로젝트 서비스 실행
```

### 4.2 주요 변경점

| 항목 | 기존 (v1.3) | 변경 (v1.4) |
|------|-------------|-------------|
| 진입점 | 랜딩 페이지 (`/`) | **커뮤니티 메인 (`/`)** |
| 앱 경로 | `/app/*` | **루트 (`/`)에서 바로** |
| 레이아웃 | 사이드바 기반 | **상단 네비게이션 기반** |
| 프로젝트 접근 | 사이드바에서 | **새 탭에서 별도 실행** |
| UI 스타일 | Linear/SaaS 스타일 | **커뮤니티 사이트 스타일** |
| 테마 | 다크 테마 기본 | **라이트 테마 기본** |

### 4.3 URL 구조 (변경)

```
FLOWIT/
├── /                           # 메인 = 커뮤니티 홈
├── /community                  # 커뮤니티 게시판
│   ├── /community/[id]         # 게시글 상세
│   └── /community/write        # 게시글 작성
├── /qna                        # Q&A 게시판
│   ├── /qna/[id]               # Q&A 상세
│   └── /qna/write              # Q&A 작성
├── /recruit                    # 팀원 모집
│   ├── /recruit/[id]           # 모집글 상세
│   └── /recruit/write          # 모집글 작성
├── /auth
│   ├── /auth/login             # 로그인
│   ├── /auth/signup            # 회원가입
│   └── /auth/callback          # OAuth 콜백
├── /mypage                     # 마이페이지
│   ├── /mypage/profile         # 프로필 설정

    ├── /projects               # 프로젝트 목록
    ├── /projects/new           # 프로젝트 생성
    └── /projects/[id]          # 프로젝트 상세 (칸반보드)

## 7. 팀 구성

### 7.1 팀원 및 역할
```
총 5명 팀

PM/인프라: 1명 (윤종호)
- 프로젝트 총괄 관리
- 인프라 설정 및 배포
- 백엔드 개발

프론트엔드 개발자: 1명 (박지승)
- Nuxt.js 개발
- UI/UX 구현

백엔드 개발자: 1명
- NestJS API 개발
- DB 설계 및 최적화

디자이너: 1명
- UI/UX 디자인
