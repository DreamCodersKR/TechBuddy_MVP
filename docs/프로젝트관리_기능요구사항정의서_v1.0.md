# 프로젝트 관리 기능 - 요구사항 정의서

**작성자**: 박지승
**작성일**: 2025-11-04
**버전**: v1.0
**문서 목적**: 부트캠프 학생용 초보자 친화적 프로젝트 관리 도구 기능 정의

---

## 1. 배경 및 목적

### 1.1 왜 이 기능이 필요한가?

#### 학생 입장의 Pain Point

**프로젝트 진행 시 방향성 부재**

- 특정 인원을 제외하고 대부분의 학생들이 자신이 **"무엇을 해야 하는지"** 명확히 알지 못함
- 코드 작성이 어려워도 산출문서조차 어디서부터 시작해야 할지 몰라 헤맴
- 충분한 예시와 따라할 수 있는 가이드라인의 부재

**팀 협업의 어려움**

- 회의를 하더라도 **"무엇을 회의해야 하는지"** 불명확
- 프로젝트의 목표와 범위가 명확하지 않아 팀원 간 내용 공유가 원활하지 않음
- 각자 다른 이해도로 작업을 진행하여 통합 시 충돌 발생

#### 강사 입장의 Pain Point

**진행 상황 파악의 어려움**

- 모든 프로젝트 산출물을 제출 시점에만 받아 확인하기 때문에 **중간 과정 모니터링 어려움**
- 학생들이 프로젝트 관련 질문을 거의 하지 않고, 질문을 받아도 맥락 파악이 어려움
- 어느 팀이 어떤 부분에서 막혀있는지 실시간으로 알 수 없음

**효율적인 멘토링 부재**

- Linear처럼 진행도를 시각적으로 확인할 수 있는 도구 부재
- 산출물에 직접 코멘트를 달아 피드백을 줄 수 있는 시스템 없음
- 회의 진행 중 즉시 이슈를 정리하고 작업으로 전환할 수 있는 도구 정보 부족

---

### 1.2 Linear/Jira와 어떻게 다른가?

#### 차별화 포인트

**1. 템플릿 및 가이드라인 중심 설계**

- 프로젝트 생성 시 **부트캠프 특화 템플릿** 제공
- Task(Issue) 작성 시 예시와 함께 단계별 가이드 제공
- "무엇을 작성해야 할지 모르겠다"는 고민 해소

**2. 초보자 친화적 UI/UX**

- 복잡한 기능 최소화, 필수 기능만 직관적으로 배치
- 버튼 3개 이하의 단순한 인터페이스
- 용어를 학생들이 이해하기 쉬운 표현으로 변경 (예: Issue → Task)

**3. 교육 과정에 최적화**

- 멘토-학생 간 명확한 권한 구분 (PM/팀장/팀원/멘토)
- 산출문서 보관 및 제출 기능 통합
- 프로젝트 종료 후 산출물 일괄 다운로드 기능

## 2. 타겟 사용자

### 2.1 주요 사용자 페르소나

#### 페르소나 1: 부트캠프 수강생 김종찬 (26세)

**배경**

- 비전공자, 프로그래밍 학습 1개월 차
- IT 업계 경력 없음
- 협업 도구 사용 경험 전무

**특징**

- 프로젝트 경험 처음
- 팀 프로젝트 협업 방법을 모름
- Linear, Jira, Notion 등의 도구를 써본 적 없음

**니즈**

- 복잡하지 않고 **직관적인 프로젝트 관리 도구**
- 단계별로 따라할 수 있는 **예시 템플릿**
- "지금 무엇을 해야 하는지" 명확히 알려주는 시스템

**사용 목적**

- 팀 프로젝트에서 자신이 맡은 역할과 작업 파악
- 팀원들과 진행 상황 공유
- 회의록 및 산출문서 정리

---

#### 페르소나 2: 부트캠프 강사 배준규 (34세)

**배경**

- 부트캠프 강사 경력 3년
- 현재 2개 반 운영 중 (각 반 4~5개 팀)
- 강의, 행정 업무, 프로젝트 멘토링 병행

**특징**

- 업무 과다로 각 팀의 세세한 진행 상황 파악 어려움
- 학생들의 질문에 즉각 대응하기 어려운 환경
- 프로젝트 종료 후 산출물 취합 및 평가에 많은 시간 소요

**니즈**

- 각 팀의 **프로젝트 진행 상황을 한눈에 파악**
- 학생들이 어디서 막혀있는지 실시간 모니터링
- 산출문서 관리 및 피드백을 **최대한 편리하게**
- 프로젝트 종료 후 산출물 일괄 다운로드

**사용 목적**

- 여러 팀의 프로젝트 진행도 동시 모니터링
- Task별 코멘트를 통한 비동기 멘토링
- 회의 시 실시간으로 이슈를 Task로 전환
- 최종 산출물 수집 및 보관

## 3. 핵심 사용자 시나리오

### 3.1 시나리오 1: 김종찬(학생) 프로젝트 진행 플로우

**프로젝트 시작**

1. 강사로부터 프로젝트 초대를 받아 팀 워크스페이스에 접속
2. 프로젝트 Overview에서 **멘토에게 전달받은 초기 기획안** 확인
3. 제공된 템플릿을 참고하여 팀 목표와 역할 분담 작성

**작업 진행** 4. 팀 회의를 통해 해야 할 일 목록(Task)을 생성

- 템플릿 가이드를 참고하여 Task 제목, 설명, 마감일 작성

5. 각 팀원에게 Task 할당
6. 자신에게 할당된 Task를 확인하고 **"In Progress"로 상태 변경**
7. 작업 중 막히는 부분 발생 시 Task를 **"Help"** 상태로 전환하고 사유 작성

**진행 상황 공유** 8. 주 1회 팀 회의에서 **칸반보드**를 보며 진행 상황 공유 9. 완료된 Task는 "Done"으로 이동 10. 멘토의 코멘트 확인 및 피드백 반영

**프로젝트 마무리** 11. 최종 산출문서를 문서 보관함에 업로드 12. 프로젝트 회고 작성

---

### 3.2 시나리오 2: 배준규(강사) 멘토링 플로우

**프로젝트 준비**

1. 프로젝트 시 각 팀별로 **프로젝트 생성**
2. 프로젝트 Overview에 **초기 기획 가이드라인** 작성
3. 각 팀의 팀원과 멘토 초대
4. 팀별 권한 설정 (PM 지정, 멘토 권한 부여)

**진행 상황 모니터링** 5. 대시보드에서 모든 팀의 **Task 진행 현황을 한눈에 확인** 6. "Help" 상태로 전환된 Task를 우선적으로 확인 7. 각 Task에 코멘트를 달아 **비동기 피드백** 제공

**회의 및 멘토링** 9. 주간 멘토링 회의 시 칸반보드를 화면 공유하며 진행 10. 회의 중 발견된 이슈를 즉시 **Task로 생성하고 할당** 11. 긴급한 작업은 우선순위 높음으로 설정

**프로젝트 종료** 12. 프로젝트 발표 후 각 팀의 **산출문서 일괄 다운로드** 13. 최종 평가 및 피드백 작성 14. 프로젝트 아카이브 (보관 모드 전환)

## 5. 상세 기능 명세

### 5.1 기능 1: 프로젝트 생성 및 관리

#### 5.1.1 기능 설명

사용자가 새로운 프로젝트를 생성하고, 팀원을 초대하며, 프로젝트의 기본 정보를 관리하는 기능입니다. 프로젝트 생성 시 자동으로 생성자는 PM(Project Manager) 권한을 부여받으며, Overview 작성을 통해 프로젝트의 목표와 초기 기획을 정리할 수 있습니다.

#### 5.1.2 주요 요소

- **프로젝트 CRUD**: 생성, 조회, 수정, 삭제
- **Overview 작성**: 프로젝트 초기 기획 및 목표 정의
- **멤버 권한 관리**: PM, 팀장, 팀원, 멘토 역할 구분
  - PM: 모든 권한 (프로젝트 설정, 멤버 관리, Task 생성/삭제)
  - 팀장: Task 생성/수정/삭제, 멤버 초대
  - 팀원: 자신의 Task 수정, 상태 변경
  - 멘토: 읽기 전용 + 코멘트 작성
- **팀원 초대**: 이메일 또는 초대 링크

#### 5.1.3 사용자 인터랙션

1. **프로젝트 생성 클릭**

   - "새 프로젝트" 버튼 클릭
   - 프로젝트명, 설명, 기간 입력
   - 템플릿 선택 (선택 사항)

2. **Overview 작성**

   - 프로젝트 목표 작성
   - 주요 기능 나열
   - 기술 스택 정의
   - 역할 분담 초안 작성

3. **팀원 초대 및 권한 부여**
   - 초대 버튼 클릭
   - 이메일 입력 또는 초대 링크 생성
   - 각 멤버의 역할 선택 (팀원/멘토)
   - 초대 전송

### 5.2 기능 2: Task 생성 및 관리

#### 5.2.1 기능 설명

프로젝트 진행에 필요한 작업(Task)을 생성하고 관리하는 기능입니다. Task는 Linear의 Issue와 유사하지만, 더 간단하고 직관적인 구조를 가집니다. 학생들이 쉽게 이해할 수 있도록 예시 템플릿을 제공하며, Task 생성 시 가이드를 따라 작성할 수 있습니다.

#### 5.2.2 주요 요소

- **Task CRUD**: 생성, 조회, 수정, 삭제
- **Task 속성**:
  - 제목 (필수)
  - 설명 (선택, 마크다운 지원)
  - 담당자 (필수)
  - 상태 (Todo / In Progress / Done / Help)
  - 우선순위 (높음 / 보통 / 낮음)
  - 마감일 (선택)
  - 라벨 (선택, 예: 프론트엔드, 백엔드, 디자인)
- **Task 템플릿**:
  - "로그인 기능 구현" 예시
  - "API 연동" 예시
  - "산출문서 작성" 예시

#### 5.2.3 사용자 인터랙션

1. **Task 생성**

   - "새 Task" 버튼 클릭
   - 템플릿 선택 또는 직접 작성
   - 제목, 설명, 담당자 입력
   - 우선순위 및 마감일 설정

2. **Task 할당**

   - Task 카드에서 담당자 클릭
   - 팀원 목록에서 선택
   - 여러 명 할당 가능

3. **Task 상태 변경**
   - 드래그앤드롭으로 칸반보드에서 이동
   - 또는 Task 상세에서 상태 드롭다운 선택
   - "Help" 상태 전환 시 사유 입력 필수

### 5.3 기능 3: 칸반보드

#### 5.3.1 기능 설명

Task의 진행 상태를 시각적으로 확인할 수 있는 칸반보드 기능입니다. Todo, In Progress, Done, Help의 4가지 상태로 Task를 분류하며, 쉽게 상태를 변경할 수 있습니다. 특히 "Help" 상태는 학생들이 막혔을 때 멘토나 강사에게 도움을 요청할 수 있는 독특한 기능입니다.

#### 5.3.2 주요 요소

- **4가지 상태 칼럼**:
  - **Todo**: 아직 시작하지 않은 작업
  - **In Progress**: 현재 진행 중인 작업
  - **Done**: 완료된 작업
  - **Help**: 도움이 필요한 작업 (부트캠프 특화 기능)
- **Task 카드 정보**:
  - 제목
  - 담당자 이름
  - 우선순위 아이콘
  - 마감일 (D-day 표시)
  - 코멘트 개수
- **정렬 옵션**:
  - 마감일 순
  - 생성일 순
  - 우선순위 순
- **필터링**:
  - 담당자별
  - 라벨별
  - 기간별

#### 5.3.3 사용자 인터랙션

1. **보드 확인**

   - 프로젝트 메인 화면에서 칸반보드 탭 선택
   - 4개 칼럼에 분류된 Task 카드 확인
   - 각 칼럼의 Task 개수 표시

2. **Task 상태 변경**

   - Task 카드를 드래그하여 다른 칼럼으로 이동
   - 또는 카드 클릭 → 상태 드롭다운 선택

3. **Help 요청**

   - Task를 "Help" 칼럼으로 이동
   - 팝업에서 어려운 점 입력 (필수)
   - 멘토 및 PM(담당강사)에게 전송

4. **Task 상세 확인**
   - Task 카드 클릭
   - 모달에서 상세 정보 및 코멘트 확인

### 5.4 기능 4: 산출문서 보관

#### 5.4.1 기능 설명

프로젝트 진행 중 생성되는 모든 산출문서 및 필요 자료를 저장하고 공유하는 기능입니다. 부트캠프에서는 프로젝트 종료 시 기획서, ERD, API 명세서, 발표 자료 등 다양한 산출물을 제출해야 하므로, 이를 체계적으로 관리할 수 있는 공간이 필요합니다.

#### 5.4.2 주요 요소

- **파일 업로드/다운로드**:
  - 드래그앤드롭 업로드
  - 파일 형식 제한 없음 (PDF, DOCX, PNG, ZIP 등)
  - 파일 크기 제한: 50MB (추후 조정 가능)
- **폴더 구조**:
  - 산출문서
  - 발표 자료
  - 기타
- **권한 관리**:
  - 팀원: 업로드/다운로드 가능
  - 멘토: 다운로드만 가능
- **일괄 다운로드**:
  - 프로젝트 종료 시 모든 파일 ZIP으로 다운로드

#### 5.4.3 사용자 인터랙션

1. **파일 업로드**

   - 산출문서 탭 선택
   - 폴더 선택 (기획서/설계서/발표 자료/기타)
   - 파일 드래그앤드롭 또는 업로드 버튼 클릭
   - 업로드 완료 시 팀원에게 알림 (선택 사항)

2. **파일 다운로드**

   - 파일 목록에서 원하는 파일 클릭
   - 다운로드 버튼 클릭
   - 또는 체크박스로 여러 파일 선택 후 일괄 다운로드

3. **파일 삭제/수정**
   - 파일 옆 메뉴(⋮) 클릭
   - 삭제 또는 이름 변경 선택
   - PM/팀장만 삭제 권한

## 6. 제외하는 기능

### 6.1 MVP에서는 하지 않을 것

#### 1. 타임 트래킹 (Time Tracking)

**이유**

- 부트캠프 학생들에게는 "얼마나 시간을 썼는지"보다 "무엇을 완료했는지"가 중요
- 타임 트래킹은 프리랜서나 에이전시에 필요한 기능
- 학생들이 시간 기록을 부담스러워할 수 있음
- Phase 4 이후 요청이 있을 경우 고려

---

#### 2. Gantt Chart

**이유**

- 초보자가 이해하기 복잡한 시각화
- 부트캠프 프로젝트는 대부분 1~2개월로 짧아 Gantt Chart의 필요성이 낮음
- 칸반보드로 충분히 진행 상황 파악 가능
- Phase 4 이후 장기 프로젝트 관리가 필요할 경우 고려

---

#### 3. 복잡한 권한 시스템

**이유**

- 부트캠프 팀은 대부분 3~5명의 소규모
- PM/팀장/팀원/멘토의 4가지 역할만으로 충분
- 세부적인 권한 설정 (Task별 권한, 모듈별 접근 제어 등)은 불필요
- 간단한 권한 관리가 초보자 친화적

---

#### 4. 고급 자동화 (Automation)

**이유**

- Jira의 자동화 워크플로우는 학생들이 설정하기 어려움
- MVP에서는 기본적인 알림만 제공
- 복잡한 자동화는 Phase 3 이후 검토

---

#### 5. 스프린트 관리 (Sprint Planning)

**이유**

- 부트캠프에서는 애자일 방법론을 깊이 다루지 않음
- 주차별 목표 설정 정도로 충분
- 스프린트 기획, 리뷰, 회고 등의 프로세스는 복잡함

---

### 7.2 성능 및 확장성 고려

**초기 목표**

- 동시 접속자: 최대 100명 (목포분원 진행 기준)
- 프로젝트 수: 최대 50개
- Task 수: 프로젝트당 최대 100개

## 8. 성공 지표

### 8.1 이 기능이 성공했다고 판단하는 기준

#### 정량적 지표

**사용자 활성도**

- 프로젝트 생성 후 **30일 이상 사용하는 팀이 70% 이상**
- 주 1회 이상 칸반보드 업데이트율 **80% 이상**
- Task 생성 후 완료율 **60% 이상**
- 산출문서 업로드 팀 비율 **90% 이상**

**사용자 만족도**

- 학생 설문조사에서 "사용하기 쉽다" 응답 **80% 이상**
- 학생 설문조사에서 "프로젝트 관리에 도움이 되었다" 응답 **75% 이상**
- 강사 설문조사에서 "진행 상황 파악이 편리하다" 응답 **85% 이상**

**기술적 지표**

- 페이지 로딩 속도 **3초 이내**
- 모바일 반응형 지원 **100%**
- 주요 기능 에러율 **1% 이하**

---

#### 정성적 지표

**학생 피드백**

- "처음 프로젝트 관리를 해봤는데 어렵지 않았다"
- "회의 때 칸반보드를 보며 이야기하니 효율적이었다"
- "멘토님이 코멘트로 피드백을 주셔서 좋았다"

**강사 피드백**

- "학생들의 진행 상황을 실시간으로 확인할 수 있어 멘토링이 수월했다"
- "산출문서를 한 곳에서 관리할 수 있어 편리했다"
- "Help 상태를 통해 막힌 팀을 바로 파악하고 도울 수 있었다"

---

#### 목표 달성 시나리오

**Phase 1 (MVP 출시 후 1개월)**

- 스마트인재개발원 목포분원 1개반 테스트
- 주요 기능 버그 수정
- 사용자 피드백 수집

**Phase 2 (출시 후 3개월)**

- 스마트인재개발원 목포분원 2개반 동시 사용 (최대 10개 팀)
- 사용자 만족도 70% 이상 달성
- 추가 기능 요청 분석

**Phase 3 (출시 후 6개월)**

- 스마트인재개발원과 파트너십 체결
- 사용자 만족도 80% 이상 달성
- 유료 플랜 검토
