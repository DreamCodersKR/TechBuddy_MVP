# FLOWIT 운영 관리 시스템 확장성 가이드

> 작성일: 2026-04-07
> 작성자: 윤종호 (CEO, DreamCoders)

---

## 1. 개요

FLOWIT은 예비창업자 단계에서 시작하여 점진적으로 운영 규모가 확장됩니다.
초기 혼자 운영하는 단계부터 팀 확장, 투자 유치 이후까지 관리 시스템도 함께 진화해야 합니다.
이 문서는 단계별 운영 관리 시스템 설계 방향과 확장성 기준을 정의합니다.

---

## 2. 사용자 역할(UserRole) 체계

### 2.1 역할 정의

| Role | 한국어 | 권한 범위 |
|------|--------|----------|
| `USER` | 일반 회원 | 서비스 이용 |
| `MODERATOR` | 콘텐츠 관리자 | 게시글/댓글 신고 처리, 콘텐츠 숨김/삭제 |
| `ADMIN` | DreamCoders 관리자 | 회원 관리, 크레딧/플랜 지급, 문의 처리, MODERATOR 권한 포함 |
| `SUPER_ADMIN` | CEO | 전체 권한 (역할 변경, 시스템 설정, ADMIN 권한 포함) |

### 2.2 권한 매트릭스

| 기능 | USER | MODERATOR | ADMIN | SUPER_ADMIN |
|------|------|-----------|-------|-------------|
| 서비스 이용 | ✅ | ✅ | ✅ | ✅ |
| 신고 처리 | ❌ | ✅ | ✅ | ✅ |
| 콘텐츠 강제 삭제 | ❌ | ✅ | ✅ | ✅ |
| 회원 플랜/크레딧 관리 | ❌ | ❌ | ✅ | ✅ |
| 고객 문의 답변 | ❌ | ❌ | ✅ | ✅ |
| 통계/지표 조회 | ❌ | ❌ | ✅ | ✅ |
| 역할 변경 | ❌ | ❌ | ❌ | ✅ |
| 시스템 설정 변경 | ❌ | ❌ | ❌ | ✅ |

## 3. 단계별 관리 시스템 진화 로드맵

### Phase 1: 베타 운영기 (현재 ~ 사업자 등록 전)
**규모**: 베타 테스터 20~50명

**구현**: FLOWIT 앱 내 `/admin` 통합
- UserRole 확장 (SUPER_ADMIN / ADMIN / MODERATOR)
- `/admin/users` 유저 관리 + 크레딧/플랜 수동 지급
- `/admin/stats` 베타 활동 지표 모니터링
- 고객 문의 모듈 (`/support`)
- 콘텐츠 신고 처리

**운영 방식**: CEO 혼자 SUPER_ADMIN으로 전체 운영

---

### Phase 2: 정식 서비스 운영기 (사업자 등록 후 ~ 팀 3명)
**규모**: MAU 100~500명

**추가 구현**:
- 결제 연동 (토스페이먼츠) → M7 이슈 재개
- 구독 관리 어드민
- 이메일 알림 시스템 (Resend)
- 운영자 계정 추가 (ADMIN 역할 부여)
- 어드민 액션 로그 (AdminLog 테이블)

**운영 방식**: CEO(SUPER_ADMIN) + 운영팀원 1~2명(ADMIN)

---

### Phase 3: 성장기 (시드 투자 후 ~ 팀 5명↑)
**규모**: MAU 1,000~5,000명

**아키텍처 전환**: 별도 Admin 앱 분리
- `admin.flowit.co` 독립 배포
- 비개발자 팀원도 사용 가능한 UI
- 세분화된 권한 관리 (팀별 접근 범위 제한)

**추가 구현**:
- 실시간 모니터링 대시보드 (Sentry, GA4 연동)
- CS 티켓 시스템 (문의 배정/협업)
- 부트캠프 B2B 파트너 관리 패널
- A/B 테스트 관리
- 공지사항/이벤트 관리 (CMS)

---

### Phase 4: 스케일업 (Series A 이후)
**규모**: MAU 10,000명↑

**고려 사항**:
- 전문 Admin 플랫폼 검토 (Retool, Metabase, 자체 구축)
- 데이터 웨어하우스 연동 (BigQuery, Redshift)
- 실시간 알림/모니터링 (PagerDuty, Slack 연동)
- 멀티 테넌시 (부트캠프별 독립 공간) 검토

## 4. 기술 아키텍처 결정사항

### 4.1 초기 통합 방식 (Phase 1~2)

```
FLOWIT Nuxt App
├── pages/
│   ├── admin/              ← Role Guard (ADMIN, SUPER_ADMIN)
│   │   ├── index.vue       ← 대시보드
│   │   ├── users/          ← 회원 관리
│   │   ├── inquiries/      ← 문의 처리
│   │   └── reports/        ← 신고 처리
│   └── support/            ← 유저 문의 작성
└── middleware/
    └── admin.ts            ← role 체크 → 미권한 시 / 리다이렉트

FLOWIT NestJS App
└── src/
    ├── admin/              ← AdminModule (ADMIN, SUPER_ADMIN 전용 API)
    ├── inquiry/            ← 문의 모듈
    └── report/             ← 신고 모듈
```

### 4.2 분리 기준 (Phase 3 전환 시점)

아래 조건 중 2개 이상 충족 시 별도 Admin 앱으로 분리 검토:
- 팀 인원 5명 이상
- MAU 1,000명 돌파
- 비개발자 운영자 합류
- 어드민 기능이 30개 이상

## 5. 고객 문의/건의사항 운영 정책

### 5.1 문의 유형 및 SLA

| 유형 | 설명 | 목표 응답 시간 |
|------|------|--------------|
| `BUG` | 서비스 버그 신고 | 24시간 이내 |
| `FEATURE` | 기능 건의 | 72시간 이내 (검토 후 회신) |
| `BETA_FEEDBACK` | 베타 테스터 피드백 | 48시간 이내 |
| `GENERAL` | 일반 문의 | 72시간 이내 |

### 5.2 베타 피드백 활용 방안
- 수집된 피드백 → Linear 이슈 자동 연동 검토
- 월별 피드백 리포트 → 투자자/심사 자료 활용
- 반복 요청 기능 → 우선순위 상향 조정
