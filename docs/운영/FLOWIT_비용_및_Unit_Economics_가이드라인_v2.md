## 2. AI 모델 라우팅 구조

### 2차원 라우팅 시스템

FLOWIT의 AI 라우팅은 **2가지 축**으로 동작함:
- **축 1 (수직): 구독 등급** → 어떤 그레이드(성능)의 모델을 쓸지 결정
- **축 2 (수평): Task 유형** → 어떤 브랜드/특화 모델을 쓸지 결정

### 모델 등급 매핑 (2026년 3월 기준)

| **심화** (10크레딧) | Sonnet 4.6 | GPT-5.4 | Gemini 2.5 Flash | Grok-3 |

## 5. Unit Economics

### 기본 지표 정의

| 지표 | 정의 | 목표값 |
|-----|------|-------|
| **ARPU** (Average Revenue Per User) | 유저 1인당 평균 월 수익 | ₩12,000 (구독 + 충전) |
| **CAC** (Customer Acquisition Cost) | 유료 유저 1명 획득 비용 | ≤ ₩30,000 |
| **LTV** (Lifetime Value) | 유저 1인이 생애 동안 내는 총 수익 | ≥ ₩120,000 |
| **Churn Rate** | 월간 이탈률 | ≤ 5% |
| **LTV:CAC Ratio** | 수익성 건강도 지표 (3 이상이면 건강) | **≥ 4:1** |
| **Payback Period** | CAC 회수 기간 | ≤ 3개월 |

### CAC 계산

**B2C (직접 마케팅)**

```

**B2C (부트캠프 POC → 유료 전환)**

```
[단순 계산]
POC 비용 (시나리오 C, 85만원) ÷ B2C 전환자 15명
= ₩850,000 ÷ 15 ≈ ₩57,000/유저

→ 단독으로 보면 CAC가 높아 보임.
  이는 POC 비용 전체를 B2C 전환자에게만 부담시켰기 때문.

[실질 CAC 산출식]
POC를 진행하면서 Enterprise 계약이 함께 성사되면,
Enterprise 수익이 POC 비용을 일부 상쇄한다.

= (POC 총비용 - POC 기간 내 Enterprise 수익) ÷ B2C 전환자 수
= (₩850,000 - ₩800,000) ÷ 15명
= ₩50,000 ÷ 15 ≈ ₩3,300/유저

전제 조건: POC 3개월(26년 6~9월) 내 Enterprise 계약 1건 성사
→ 이 조건이 충족되면 B2C 유저 1명 획득 실질 비용 = ₩3,300
```

**B2B (Enterprise 영업)**

```

## 11. Free 유저 크레딧 과잉 지급의 Unit Economics 영향

### 11-1. 비용 관점: 직접적 적자 위험은 낮음

Free 유저는 **기본 AI(1크레딧/메시지)만 사용 가능**하며, 기본 AI 기본값은 Gemini 2.5 Flash-Lite (₩0.4원/메시지).

```
헤비 Free 유저 월 크레딧 사용 분석 (650 크레딧 획득 가정):
  크레딧 전량 기본 AI 사용: 650 메시지 × ₩0.4 = ₩260원/월

  → AI API 비용 관점에서 "적자 없음" ✅
  → 기존 Free 유저 AI API 비용 가정 (₩700,000/월 고정비) 내에서 처리 가능
```

> **결론: M10 크레딧 보상이 직접적인 AI API 비용 증가를 유발하지는 않는다.**
> 단, 아래 수익성 지표들에 간접적으로 부정적 영향을 미친다.

---

### 11-2. 수익 관점: Pro 전환율 희석 (핵심 문제)

v1.3 비즈니스 모델의 핵심 가정: **Free 유저는 크레딧 부족으로 Pro로 전환한다.**

```
기존 설계 (M10 이전):
  Free 유저 월 크레딧: 100 cr
  AI 멘토링 사용: 기본 AI 1cr × 100 = 100 메시지/월
  → 100 메시지 소진 후 "크레딧 충전" 또는 "Pro 업그레이드" 팝업 노출
  → 유료 전환 인센티브: 강함

M10 이후 (헤비 유저 기준):
  Free 유저 월 크레딧: 650 cr
  AI 멘토링 가능 횟수: 650 메시지/월
  → 크레딧 부족 시점: 현저히 늦어짐 또는 월 내 소진 안 됨
  → 유료 전환 인센티브: 희석됨
```

#### Free → Pro 전환 트리거 분석

| 전환 트리거 | M10 이전 | M10 이후 (헤비 유저) | 영향 |
|-----------|---------|-----------------|------|
| 크레딧 소진 | 빠름 (100cr) | 느림 (650cr) | 전환 지연 🔴 |
| 심화 AI 욕구 | 발생 시점: 빠름 | 발생 시점: 변화 없음 | 중립 |
| PM툴 기능 필요 | 변화 없음 | 변화 없음 | 중립 |
| **순 전환율 예상** | **10~15%** | **7~10%** | **3~5%p 하락** ⚠️ |

#### 전환율 하락이 MRR에 미치는 영향 (3,000명 총 가입자 기준)

```
기존 설계 (전환율 10%):
  유료 유저 300명 × ₩19,980 블렌디드 ARPU (Pro 80% + Premium 20%) = ₩5,994,000 MRR

M10 이후 예상 (전환율 7%):
  유료 유저 210명 × ₩19,980 ARPU = ₩4,195,800 MRR

MRR 차이: ₩1,798,200/월 (약 30% 감소)
연간 차이: ₩21,578,400 손실 ← 유의미한 수준
```

### 11-3. 크레딧 충전 수익 감소

Free 유저가 크레딧을 소진하지 않으면 추가 충전 팝업도 적게 노출된다.

```

  크레딧 소진 시 충전 유도
  예상 충전율: 5% × ₩5,500 = ₩275/Free 유저/월

M10 이후 (크레딧 넉넉):
  충전 유도 팝업 노출 감소
  예상 충전율: 2% × ₩5,500 = ₩110/Free 유저/월

  감소분: ₩165/Free 유저/월 × 2,700명(비유료) = ₩445,500/월 손실
```

> 30일 스트릭에 Pro 체험권 부여 → 심화 AI/PM툴 전체 기능 경험 → 유료 전환 유도.

### 13-6. 체험권(Trial) 구현 가이드라인

Pro 체험권은 기존 구독 시스템을 임시 승격하는 방식으로 구현한다.

**체험권에 크레딧 함께 지급 여부**

기능만 개방할 경우, Free 유저 보유 크레딧(권고안 기준 ~50~100 cr)으로는 심화 AI를 5~10번밖에 체험할 수 없어 Pro의 가치를 충분히 경험하기 어렵다.

| 방식 | 체험 가능 횟수 | 전환 설득력 |
|------|------------|----------|
| 기능만 개방 (크레딧 없음) | 심화 AI 5~10번 | 낮음 |
| 일할계산 크레딧 (700 cr) | 심화 AI 70번 | 과잉 |
| **체험 전용 크레딧 300 cr** | 심화 AI 30번 | 최적 ✅ |

> **권고**: Pro 7일 체험권 + **체험 전용 크레딧 300 cr** 동시 지급
> - 심화 AI 30번 사용 가능 → 충분한 체험 후 전환 유도
> - 체험 크레딧은 **일반 크레딧과 별도 필드로 관리** → Free 크레딧 경제에 영향 없음
> - 체험 만료 시 **미사용 체험 크레딧 자동 소멸** (일반 크레딧 잔액에 합산 안 됨)

**DB 스키마 변경 (권고)**

```prisma
model TrialGrant {
  id               String   @id @default(cuid())
  userId           String
  plan             UserPlan // PRO or PREMIUM
  startsAt         DateTime
  endsAt           DateTime
  reason           String   // "STREAK_30", "STREAK_90", "REFERRAL_5"
  trialCredits     Int      @default(300) // 체험 전용 크레딧 (일반 크레딧과 별도)
  trialCreditsUsed Int      @default(0)
  used             Boolean  @default(false)
  createdAt        DateTime @default(now())

  user             User     @relation(fields: [userId], references: [id])
}
```

**체험권 동작 원칙**
- 체험권 발급 시 `endsAt` = 현재 + 체험 기간
- AI 사용 시 체험 크레딧 우선 차감 → 소진 후 일반 크레딧 차감
- 유료 구독이 이미 있는 경우: 체험권 발급하되 구독 종료 후 자동 적용
- 체험 기간 중 구독 취소 시: 체험권도 즉시 종료, 미사용 체험 크레딧 소멸
- 체험 만료 D-3, D-1: 인앱 알림 + 이메일 발송 ("남은 체험 크레딧 XXX개, Pro 유지하기 →")

## 결론

M10 게이미피케이션은 **리텐션(DAU/재방문율) 향상에는 효과적**이나,
현행 크레딧 보상 구조는 **Free → Pro 전환 인센티브를 희석**시켜 Unit Economics에 부정적 영향을 줄 수 있다.

**API 비용 관점에서는 문제 없음** (Free = 기본 AI만, 0.4원/메시지).
**수익 관점에서 핵심 위험**: Pro 전환율 8%대 고착 시 3,000명 기준 월 BEP(313명) 미달.

**즉시 시행 권고 사항 2가지:**
1. 데일리 퀘스트 크레딧 보상 → XP 전환 (P0)
2. 30일 스트릭 100cr → Pro 7일 체험권 전환 (P0)

위 두 가지만 적용해도 Free 유저 월 크레딧 획득량이 650cr → 100cr 수준으로 정상화되며,
Pro 체험권을 통한 업셀링 퍼널이 활성화되어 전환율 12% 회복 시 BEP 달성(+₩861,088 흑자)이 가능하다.

> **v2.1 참고**: Pro:Premium 80:20 기준에서는 시나리오 Y(전환율 10%)도 BEP에 소폭 미달하므로,
> 권고안(시나리오 Z) 적용을 통한 전환율 12% 확보가 더욱 중요해졌다.
